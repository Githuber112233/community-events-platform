## 后端问题修复

### 修复的安全漏洞
| 文件 | 问题 | 修复 |
|------|------|------|
| `AdminController.isAdmin()` | **任何登录用户都能访问管理员接口**（只检查userId非null） | 改为查询数据库验证 UserRole.ADMIN |
| `CommentController.extractUserIdFromToken()` | **JWT解析未实现**，永远返回 1L | 正确调用 jwtUtil.getUserIdFromToken(token) |

### 修复的 Spring Data JPA 方法名
| Repository | 旧方法名 | 新方法名 |
|-----------|---------|---------|
| `ActivityParticipantRepository` | `findByUserIdAndActivityId` | `findByUser_IdAndActivity_Id` |
| `ActivityParticipantRepository` | `existsByUserIdAndActivityId` | `existsByUser_IdAndActivity_Id` |
| `ActivityLikeRepository` | 同上 | 同上 |

### 修复的 SecurityConfig 路径问题
- **严重**：`requestMatchers` 路径错误加了 `/api` 前缀
- Spring Security 的 `requestMatchers` 在有 `context-path: /api` 时本身就是相对于 servlet path，去掉所有 `/api` 前缀
- 影响：之前所有公开接口都返回 401

### 新增接口
- `POST /auth/admin/login` - 管理员登录（与普通登录共用，需 ADMIN 角色）
- `POST /users/{userId}/follow` - 关注用户（前端 followApi 路径）
- `DELETE /users/{userId}/follow` - 取消关注
- `GET /users/{userId}/followers` - 粉丝列表
- `GET /users/{userId}/following` - 关注列表

### 修复的前端 API 路径
- `userApi.updateUserInfo` 从 `/users/profile` 修正为 `/users/me`

### Lombok 注解处理器
- `pom.xml` 显式配置 `maven-compiler-plugin` 的 `annotationProcessorPaths`，避免 Lombok 不生成 getter/setter

### 数据库 schema 修复
